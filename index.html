Case Study

Evidence-Based Course Review for Access, Clarity, and Learner Success

Executive Summary

This project is a self-paced course for faculty, instructional designers, and academic partners in higher education. It teaches a repeatable process for reviewing course artifacts, interpreting evidence across multiple dimensions, and prioritizing the changes that most improve clarity, access, and learner support. A consistent module and milestone structure helps make instructional judgment teachable.

Although the course has not yet run with learners, it is designed to improve consistency in institutional course-review practices, make reviewer reasoning more visible, and support clearer, more defensible faculty-facing recommendations.

Opening Positioning Statement

I design scalable learning systems that help educators make course improvement work clear, structured, and usable.

This project is a self-paced course for faculty, instructional designers, and academic partners. It teaches an evidence-based process for reviewing course design, identifying barriers, evaluating priorities, and writing practical, faculty-facing recommendations.

I designed the course as a complete learning system rather than a set of content pages. A consistent module structure, milestone outputs, and embedded supports guide learners from noticing course features to developing evidence-based recommendations.

The result is a cohesive experience in which the architecture, milestone logic, supports, and final performance all reinforce the same reasoning process.

What This Project Demonstrates

Teaches a judgment skill that many courses leave implicit.

Course review is often treated like a checklist task. This course makes reasoning visible, teachable, and transferable without instructor mediation.

Backward design applied at the system level.

Every module, milestone, and support was built in service of a single authentic performance task defined before any content was written.

The Challenge

Learners needed to distinguish evidence from assumption, evaluate course quality across five dimensions, prioritize improvements, and write faculty-facing recommendations without instructor mediation.

At the same time, the course needed to reduce ambiguity while preserving authentic analytical work and maintaining a focus on course design rather than learner-deficit assumptions.

Design Strategy

I structured the course around a visible reasoning sequence that guides learners from initial observation to faculty-facing recommendation.

Observation → Evidence → Interpretation → Prioritization → Recommendation

The final performance task was defined first, and every module, milestone, and support was built in service of that outcome.

Learning Architecture

The course is organized as a five-module progression aligned to the reasoning sequence, moving from observation to recommendation.

Each module produces a working artifact that feeds forward into the next stage, making the reasoning process visible and cumulative.

A consistent module pattern reduces navigation burden and supports progression toward independent judgment.

Scaffolding, Accessibility, and Inclusion

Support is embedded directly into the workflow through tools, comparisons, and quality checks at points of highest cognitive demand.

Structure, terminology, and criteria are consistent and visible so learners understand expectations without inference.

Support fades across modules as learners move toward independent performance.

Key Design Decisions

Each decision supports clarity, usability, and transfer.

1. Consistent module structure

Reduces navigation load and keeps attention on reasoning.

2. Visible reasoning sequence

Clarifies expectations and supports transfer.

3. Point-of-use support

Reduces ambiguity without lowering rigor.

4. Judgment-based final task

Emphasizes reasoning over surface polish.

Authentic Performance Design

The final task mirrors real course review work through multi-artifact analysis, prioritization, and recommendation writing.

Supporting artifacts make reasoning visible and defensible.

Outcome and Reflection

This project creates a structured, repeatable course review process.

It demonstrates how complex instructional judgment can be made teachable in a self-paced environment.

Future refinement will focus on visual signaling and transition clarity.
